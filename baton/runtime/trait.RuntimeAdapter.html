Expand description

Interface for agent runtime backends.

Implementations manage the full session lifecycle: creation, polling, result collection, cancellation, and cleanup.

§Session lifecycle

  1. health_check — verify the runtime is reachable
  2. create_session — start an agent session
  3. poll_status — wait for completion
  4. collect_result — retrieve the output
  5. teardown — clean up resources

For one-shot queries without a session, use post_completion instead.
